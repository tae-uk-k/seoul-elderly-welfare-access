1단계. 데이터 품질 검증 (복지_데드존_분석_메인스크립트.py)
------------------------------------------------------------

STEP 1 — 파일별 품질 체크
- 각 파일의 행수·결측률·중복 수 자동 계산
- 자치구 25개 커버 여부, 좌표 컬럼 유무 확인
- 노인복지시설: 전 25개 자치구 커버, 주소 결측 거의 없음, 좌표 없음 확인
- 독거노인: 구명 forward-fill 후 행정동 행 426건 파싱 완료
- 버스·지하철: 서울 범위(lat 37.4~37.7, lon 126.8~127.2) 필터링

STEP 2 — 결합 가능성 검증
- 코드 형식 불일치 문제 정리: 65세인구(10자리), 생활인구(5자리), 독거노인(동명), 격자(GID)
- 버스정류소 ↔ 격자 공간조인 테스트: WGS84 → EPSG:5179 변환 후 sjoin 실행 성공

결론: 행정동 단위 완전 분석을 위해 ①행정동 경계 SHP, ②행정동 단위 65세+ 인구 추가 필요


2단계. 노인복지시설 지오코딩 (지오코딩_주소변환.py)
-----------------------------------------------------

1차 지오코딩 (전체 3,126건)

1. 헤더 처리: 원본 엑셀은 row0=타이틀, row1=헤더 구조 → iloc[2:]로 데이터 추출
2. 주소 정제:
   - 괄호 안 동명 제거: (삼성동) → 삭제
   - 중복 공백 제거
   - '서울특별시' 접두어 미포함 시 자동 추가
3. 카카오 로컬 API 호출: 요청 간격 0.1초, timeout 10초
4. Fallback 전략: NO_RESULT 시 '서울특별시 {자치구} {시설명}' 으로 재시도
5. RATE_LIMIT 처리: 0.5초 대기 후 자동 재시도
6. 서울 범위 검수: 좌표 범위 이탈 건 OUT_OF_RANGE 처리 후 좌표 제거
7. 출력: 서울_노인복지시설_좌표.xlsx, 지오코딩_실패목록.xlsx

2차 재시도 (실패 204건, 지오코딩_실패주소_재시도.py)

주소 정제 규칙을 대폭 강화:

  대괄호 동명 제거:   홍은제1동[홍은동] → 홍은제1동
  층수 제거:         지1,2~4층 / 3,4층 / B5층 등 복합 패턴 모두 처리
  호수 제거:         동N호 / 번지뒤 호수 / 복합 A-B호 / 3자리이상 단독호
  마침표 비정상:     .장로.2길 → 장로.2길
  숫자만인 오염 데이터: INVALID_DATA로 스킵

- 여전히 실패 시 구+시설명 fallback 재시도
- 성공 건은 메인 파일(서울_노인복지시설_좌표.xlsx)에 시설명+자치구+주소 3중 매칭으로 업데이트
- 최종 실패 건 → 지오코딩_최종실패.xlsx 별도 보관


3단계. 독거노인 행정동 코드 매핑
----------------------------------

- 원본 독거노인 파일은 동명 텍스트만 존재
- 행정동 SHP(BND_ADM_DONG_PG.shp)의 ADM_NM과 특수문자 제거 후 텍스트 매핑
- 결과: 독거노인_행정동코드포함.csv 생성 (행정동코드 컬럼 추가)


4단계. 행정동별 65세이상인구 집계
------------------------------------

- 연령별 인구 파일에서 65세~100세이상 컬럼 합산
- 결과: 행정동별_65세이상인구.csv 생성


5단계. 사전 분석 (자치구 단위, n=25)
---------------------------------------

- 접근성 지표: 구 중심점 거리 무의미(25개 구 모두 복지관 1개 이상) → 1/복지관밀도 로 대체
- 데드존 후보: 관악구(65세+ 93,239명), 구로구(65세+ 87,137명)
- 상관관계: 65세인구↔독거노인 r=0.812(강함), 생활인구↔접근성 r=-0.003(약함, 구 단위 한계)
- 그래프 3종 생성: 접근성부족지수 막대, 상관관계 히트맵, 시설집중도 불균형
